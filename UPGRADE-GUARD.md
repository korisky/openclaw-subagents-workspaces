# OpenClaw Upgrade Guard — JARVIS Multi-Agent Protection

## What's At Risk During Upgrades

| Component | Risk Level | Location | Protected? |
|-----------|-----------|----------|------------|
| Workspace files (SOUL.md, AGENTS.md) | LOW | GitHub repo | Yes — git tracked |
| openclaw.json (multi-agent config) | **HIGH** | ~/.openclaw/openclaw.json | Needs backup |
| Systemd service (proxy env vars) | **HIGH** | ~/.config/systemd/user/openclaw-gateway.service | Needs backup |
| Agent auth credentials | MEDIUM | ~/.openclaw/agents/{jarvis,code}/agent/ | Needs backup |
| Discord intents (Developer Portal) | NONE | Discord cloud | Not affected by local upgrade |

## Pre-Upgrade Checklist

Run this BEFORE any `openclaw update`:

```bash
bash ~/.openclaw/openclaw-subagents-workspaces/scripts/pre-upgrade.sh
```

## Post-Upgrade Checklist

Run this AFTER any `openclaw update`:

```bash
bash ~/.openclaw/openclaw-subagents-workspaces/scripts/post-upgrade.sh
```

## Manual Recovery

If the upgrade broke things and scripts aren't available:

### 1. Restore openclaw.json
```bash
# Find latest backup
ls -lt ~/.openclaw/openclaw.json.pre-upgrade.* | head -1

# Restore
cp ~/.openclaw/openclaw.json.pre-upgrade.<timestamp> ~/.openclaw/openclaw.json
```

### 2. Restore systemd service proxy
```bash
# Check if proxy vars are present
grep -c "HTTP_PROXY" ~/.config/systemd/user/openclaw-gateway.service

# If 0, add them back:
# Environment=HTTP_PROXY=http://127.0.0.1:7897
# Environment=HTTPS_PROXY=http://127.0.0.1:7897
# Environment=ALL_PROXY=http://127.0.0.1:7897
# Environment=NO_PROXY=localhost,127.0.0.1,::1

systemctl --user daemon-reload
systemctl --user restart openclaw-gateway
```

### 3. Verify agents exist
```bash
openclaw doctor
# Should show: Agents: jarvis (default), code
# If not, restore openclaw.json from backup
```

### 4. Verify workspace paths
```bash
# Workspaces should still point to the git repo
grep "workspace" ~/.openclaw/openclaw.json
# Expected paths:
# /home/roylic/.openclaw/openclaw-subagents-workspaces/workspace-jarvis
# /home/roylic/.openclaw/openclaw-subagents-workspaces/workspace-code
```

## Critical Config Sections to Verify After Upgrade

These sections are custom (not generated by OpenClaw wizard) and may be lost:

1. `agents.list` — the jarvis + code agent definitions
2. `tools.agentToAgent` — must exist
3. `tools.sessions.visibility` — must be "tree"
4. `bindings` — maps discord to jarvis
5. `channels.discord.threadBindings` — sub-agent thread support
6. `agents.defaults.subagents` — timeout and archive settings
7. `agents.defaults.memorySearch` — hybrid search config
